# Content Pillars

## 1. Agent Memory Thesis

ClipBrain exists because agents are powerful but still start cold. They need
access to the user's saved reading, highlights, PDFs, videos, and threads with
citations and source boundaries.

## 2. Build Logs

Daily, concrete progress from the repo: diagnostics, retrieval quality, setup
fixes, context packs, first-run polish, capture reliability, and safety checks.

## 3. Product Proof

Demos that show the loop: clip something, retrieve it, hand it to an agent, and
get a source-grounded answer. Screenshots and short videos beat abstract claims.

## 4. Local-First Trust

ClipBrain is not a hosted memory cloud. It runs locally, keeps the server on
loopback by default, and uses optional AI enrichment only when configured.

## 5. Reader Workflows

Kindle highlights, web articles, PDFs, Gmail threads, YouTube transcripts, and
Obsidian sync are not feature checkboxes. They are entry points into a living
agent memory workflow.

## 6. Taste And Standards

The public story should show the bar: no corporate fluff, no silent setup
failure, no fake magic. If it is not reliable enough for a stranger to feel in
ten seconds, keep polishing.
